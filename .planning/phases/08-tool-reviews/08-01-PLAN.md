Each tool review dict contains:

- `gtm_use_cases`: HTML with 4-6 specific use cases for GTM Engineers. Not generic sales use cases. Think: "Waterfall enrichment across 10+ data providers in a single table", "Automated ICP scoring with Clay formulas + OpenAI columns", etc.
- `pricing`: HTML table or structured breakdown of actual pricing tiers. Include free tier if exists. Note what's NOT included (common gotchas like credit limits, seat minimums, overage costs). For tools with opaque pricing (ZoomInfo, 6sense), say so directly.

This section is what differentiates the reviews from vendor marketing.
- `verdict`: 1-2 paragraphs. Clear recommendation: who should use it, who shouldn't, and what alternatives to consider. Confident opinion.
- `faq`: List of 3-5 (question, answer) tuples. Real questions GTM Engineers ask. Not fluff.
